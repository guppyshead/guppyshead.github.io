HashMap源码解析(JDK1.8)

HashMap JDK
JDK源码
发布日期:   2021-05-07
更新日期:   2021-05-08

数据结构

在JDK1.8中，HashMap的数据结构为哈希桶数组，哈希桶数组为数组+链表，每个链表的头节点为数组重的一个元素，当链表中元素个数达到8个，并且数组长度大于64时，会将链表转化为红黑树。

主要成员变量及含义

//默认数组大小，如果不指定则为16
static final int DEFAULT_INITIAL_CAPACITY = 1 << 4; // aka 16
//数组最大长度，数值为2^30，当数组长度达到这个值时，数组不会再进行扩容
static final int MAXIMUM_CAPACITY = 1 << 30;
//默认负载因子
static final float DEFAULT_LOAD_FACTOR = 0.75f;
//链表转为红黑树时，链表最少应包含的元素个数
static final int TREEIFY_THRESHOLD = 8;
//哈希桶数组扩容后，如果红黑树中元素个数少于6个，将会退化成链表
static final int UNTREEIFY_THRESHOLD = 6;
//链表转化为红黑树时，数组的最小长度（如果数组长度小于64，则会进行扩容而不是将链表转化为红黑树）
static final int MIN_TREEIFY_CAPACITY = 64;
//哈希桶数组。
transient Node<K,V>[] table;
//当HashMap中包含多少个元素时，将进行扩容
int threshold;
//实际的负载因子
final float loadFactor;

HashMap的哈希桶数组使用transient关键字修饰。其原因主要是对于不同的虚拟机，同一个对象的hashCode的值并不一定一样，也就是对于同一个HashMap，在不同的虚拟机中，同一个元素所在的哈希桶数组的位置可能不一样；因此序列化的时候不保存哈希桶数组而只是保存HashMap中所有的元素，在反序列化的时候将所有元素取出，然后重新构建哈希桶数组。

不直接使用红黑树存储数据是因为树节点占用的空间大概为链表节点的两倍，而且红黑树插入元素时操作更加复杂。当链表长度为8的时候转化为红黑树是因为根据柏松分布，如果负载因子为0.75时，λ大概为0.5，这种情况下每个链表长度出现的概率为

链表长度 出现概率
0 0.60653066
1 0.30326533
2 0.07581633
3 0.01263606
4 0.00157952
5 0.00015795
6 0.00001316
7 0.00000094
8 0.00000006

可以看到，链表长度为8出现的概率很小，因此这个时候转化为红黑树性价比很高

构造函数

无参构造

public HashMap() {
  this.loadFactor = DEFAULT_LOAD_FACTOR;
}

无参构造函数只是将负载因子进行赋值，并没有构造哈希桶数组

有参构造

public HashMap(int initialCapacity, float loadFactor) {
  if (initialCapacity < 0)
      throw new IllegalArgumentException("Illegal initial capacity: " +
                                          initialCapacity);
  //如果传入的初始数组大小大于最大长度，则去最大长度
  if (initialCapacity > MAXIMUM_CAPACITY)
      initialCapacity = MAXIMUM_CAPACITY;
  if (loadFactor <= 0 || Float.isNaN(loadFactor))
      throw new IllegalArgumentException("Illegal load factor: " +
                                          loadFactor);
  this.loadFactor = loadFactor;
  this.threshold = tableSizeFor(initialCapacity);
}

有参构造函数中对负载因子进行赋值，并根据入参中的初始长度生成threshold（此时的threshold等于数组长度，但在真正构建数组的时候会变为数组长度*负载因子）
tableSizeFor方法代码如下

static final int tableSizeFor(int cap) {
  int n = cap - 1;
  //保证n用二进制表示时最高2位为1（这里的最高位指二进制表示时从左到右数，第一个不为0的位置，如果n用二进制表示时小于等于2位，则从最高位起每一位都已经被改为1，以下同理）
  n |= n >>> 1;
  //保证n用二进制表示时最高4位为1
  n |= n >>> 2;
  //保证n用二进制表示时最高8位为1
  n |= n >>> 4;
  //保证n用二进制表示时最高16位为1
  n |= n >>> 8;
  //保证n用二进制表示时最高32位为1
  n |= n >>> 16;
  return (n < 0) ? 1 : (n >= MAXIMUM_CAPACITY) ? MAXIMUM_CAPACITY : n + 1;
}

可以看到，tableSizeFor中在return前的代码保证了n必定为2的幂次方-1，所以这个方法返回的数组长度必定为2的幂次方

添加元素

HashMap添加元素的方法如下

public V put(K key, V value) {
  return putVal(hash(key), key, value, false, true);
}

hash方法中根据key的hashCode进行了一次hash算法，这种hash算法，将hashCode的高16位和低16位都进行了运算，可以有效减少哈希碰撞，实现如下

static final int hash(Object key) {
  int h;
  return (key == null) ? 0 : (h = key.hashCode()) ^ (h >>> 16);
}

真正放入元素的方法为putVal，实现如下

final V putVal(int hash, K key, V value, boolean onlyIfAbsent,
               boolean evict) {
  Node<K,V>[] tab; Node<K,V> p; int n, i;
  //初始化哈希桶数组
  if ((tab = table) == null || (n = tab.length) == 0)
      n = (tab = resize()).length;
  //如果所在位置的数组元素为空，则直接放入数组该位置
  if ((p = tab[i = (n - 1) & hash]) == null)
      tab[i] = newNode(hash, key, value, null);
  else {
    Node<K,V> e; K k;
    //这里的e用来保存根据key找到的node，如果不存在，则e为null
    if (p.hash == hash &&
      ((k = p.key) == key || (key != null && key.equals(k))))
      e = p;
    else if (p instanceof TreeNode)
      //如果数组的第一个元素是树形节点，则使用树形节点的putTreeVal方法
      e = ((TreeNode<K,V>)p).putTreeVal(this, tab, hash, key, value);
    else {
      //到这一步就说明数组中该位置为链表
      for (int binCount = 0; ; ++binCount) {
          //遍历链表，如果直到尾部都没有该元素，则在链表尾部新增元素，并判断是否要转变为红黑树
        if ((e = p.next) == null) {
            p.next = newNode(hash, key, value, null);
            if (binCount >= TREEIFY_THRESHOLD - 1)
                treeifyBin(tab, hash);
            break;
        }
        if (e.hash == hash &&
            ((k = e.key) == key || (key != null && key.equals(k))))
            break;
        p = e;
      }
    }
    //如果e不为null，则说明在HashMap中存在put的元素，则将改变该key的value，并返回之前的value
    if (e != null) {
      V oldValue = e.value;
      if (!onlyIfAbsent || oldValue == null)
          e.value = value;
      //这是个空实现，主要为了LinkedHashMap向双向链表中插入元素
      afterNodeAccess(e);
      return oldValue;
    }
  }
  ++modCount;
  //如果HashMap中的元素个数超过最大限度，则哈希桶数组进行扩容
  if (++size > threshold)
    resize();
  //这是个空实现，主要为了LinkedHashMap根据规则删除node节点（例如LRU删除最早插入的节点）
  afterNodeInsertion(evict);
  return null;
}

putVal的主要逻辑如下：

  1. 对哈希桶数组进行判断，如果哈希桶数组还未初始化，则调用resize方法进行初始化;
  2. 将hash值对数组长度进行取模，获取插入元素在数组中的位置，如果数组该位置为空，则直接构造元素，并放入数组对应位置；
  3. 如果数组该位置有值，则在链表（或红黑树）中寻找是否key已经存在，如果存在则修改value，并返回之前的value，如果不存在，则插入链表（或红黑树）中，并判断是否要将链表转化为红黑树；
  4. 判断元素插入后是否需要扩容，如果需要，则调用resize方法进行扩容

向链表中插入元素的时候，使用的是尾插法，而不是像JDK1.7中一样使用头插法。主要是因为在JDK1.7中，多线程环境下，如果哈希桶的扩容和元素插入同时进行，有可能会形成环形链表，导致get和put元素时会一直在环形链表中死循环。

putVal中初始化和扩容时都调用了resize方法，其实现如下

final Node<K,V>[] resize() {
    Node<K,V>[] oldTab = table;
    int oldCap = (oldTab == null) ? 0 : oldTab.length;
    int oldThr = threshold;
    int newCap, newThr = 0;
    //如果老数组的长度大于0，则扩容
    if (oldCap > 0) {
      //如果数组已经达到最大长度，则不进行扩容，直接返回当前数组
      if (oldCap >= MAXIMUM_CAPACITY) {
        threshold = Integer.MAX_VALUE;
        return oldTab;
      }
      //否则将数组长度和threshold翻倍
      else if ((newCap = oldCap << 1) < MAXIMUM_CAPACITY &&
             oldCap >= DEFAULT_INITIAL_CAPACITY)
        newThr = oldThr << 1;
    }
    else if (oldThr > 0)
      //这里已经确定是数组还未初始化，如果threshold大于0，则说明HashMap是根据有参构造生成的，那么此时的threshold即为数组初始化长度
      newCap = oldThr;
    else {
      //这里确定数组未初始化，并且是HashMap根据无参构造生成的，因此直接使用默认值即可
      newCap = DEFAULT_INITIAL_CAPACITY;
      newThr = (int)(DEFAULT_LOAD_FACTOR * DEFAULT_INITIAL_CAPACITY);
    }
    //如果HashMap是根据有参构造生成的，newThr并未被赋值，要重新计算threshold的值
    if (newThr == 0) {
      float ft = (float)newCap * loadFactor;
      newThr = (newCap < MAXIMUM_CAPACITY && ft < (float)MAXIMUM_CAPACITY ?
                (int)ft : Integer.MAX_VALUE);
    }
    threshold = newThr;
    @SuppressWarnings({"rawtypes","unchecked"})
        Node<K,V>[] newTab = (Node<K,V>[])new Node[newCap];
    table = newTab;
    //对扩容后的数组进行填充
    if (oldTab != null) {
      for (int j = 0; j < oldCap; ++j) {
        Node<K,V> e;
        //对数组中每个元素的处理，要注意的是，如果老数组长度为n，元素所处的数组位置为i，则扩容后，这个元素所处的数组位置要么是i，要么是i+n
        if ((e = oldTab[j]) != null) {
          oldTab[j] = null;
          if (e.next == null)
            newTab[e.hash & (newCap - 1)] = e;
          else if (e instanceof TreeNode)
            //对于树节点的处理，方法中判断了重新生成的两棵树是否元素个数大于6，如果不是的话，则红黑树会退化成链表
            ((TreeNode<K,V>)e).split(this, newTab, j, oldCap);
          else {
            //对于链表的处理
            Node<K,V> loHead = null, loTail = null;
            Node<K,V> hiHead = null, hiTail = null;
            Node<K,V> next;
            do {
              next = e.next;
              //如果hash与oldCap做与运算结果为0，则说明元素所处的数组位置不用改变。（只有oldCap为2^n时才能这么判断）
              if ((e.hash & oldCap) == 0) {
                if (loTail == null)
                  loHead = e;
                else
                  loTail.next = e;
                loTail = e;
              }
              else {
                if (hiTail == null)
                  hiHead = e;
                else
                  hiTail.next = e;
                hiTail = e;
              }
            } while ((e = next) != null);
            if (loTail != null) {
              loTail.next = null;
              newTab[j] = loHead;
            }
            if (hiTail != null) {
              hiTail.next = null;
              newTab[j + oldCap] = hiHead;
            }
          }
        }
      }
    }
    return newTab;
}

resize方法中扩容时，对树形节点调用了树节点的split方法，其实现如下

final void split(HashMap<K,V> map, Node<K,V>[] tab, int index, int bit) {
  TreeNode<K,V> b = this;
  TreeNode<K,V> loHead = null, loTail = null;
  TreeNode<K,V> hiHead = null, hiTail = null;
  int lc = 0, hc = 0;
  //将一颗红黑树变为两颗红黑树
  for (TreeNode<K,V> e = b, next; e != null; e = next) {
    next = (TreeNode<K,V>)e.next;
    e.next = null;
    //如果hash与oldCap做与运算结果为0，则说明元素所处的数组位置不用改变。
    if ((e.hash & bit) == 0) {
      if ((e.prev = loTail) == null)
        loHead = e;
      else
        loTail.next = e;
      loTail = e;
      ++lc;
    }
    else {
      if ((e.prev = hiTail) == null)
        hiHead = e;
      else
        hiTail.next = e;
      hiTail = e;
      ++hc;
    }
  }

  if (loHead != null) {
    //判断是否需要将红黑树转换为链表
    if (lc <= UNTREEIFY_THRESHOLD)
      tab[index] = loHead.untreeify(map);
    else {
      tab[index] = loHead;
      if (hiHead != null)
        loHead.treeify(tab);
    }
  }
  if (hiHead != null) {
    //同上
    if (hc <= UNTREEIFY_THRESHOLD)
      tab[index + bit] = hiHead.untreeify(map);
    else {
      tab[index + bit] = hiHead;
      if (loHead != null)
        hiHead.treeify(tab);
      }
  }
}

在putVal方法中，构造红黑树调用了treeifyBin方法，其实现如下

  final void treeifyBin(Node<K,V>[] tab, int hash) {
    int n, index; Node<K,V> e;
    //如果哈希桶数组为空，或者数组长度小于64，先扩容，而不是转化为红黑树
    if (tab == null || (n = tab.length) < MIN_TREEIFY_CAPACITY)
      resize();
    else if ((e = tab[index = (n - 1) & hash]) != null) {
      TreeNode<K,V> hd = null, tl = null;
      do {
        TreeNode<K,V> p = replacementTreeNode(e, null);
        if (tl == null)
          hd = p;
        else {
          p.prev = tl;
          tl.next = p;
        }
        tl = p;
      } while ((e = e.next) != null);
      if ((tab[index] = hd) != null)
        hd.treeify(tab);
  }
}

获取元素

获取元素使用的是get方法，实现如下

public V get(Object key) {
  Node<K,V> e;
  return (e = getNode(hash(key), key)) == null ? null : e.value;
}

实际是根据getNode方法来获取元素的

final Node<K,V> getNode(int hash, Object key) {
  Node<K,V>[] tab; Node<K,V> first, e; int n; K k;
  //先根据hash算出所在数组下标，然后遍历链表（红黑树）寻找hashCode等于key并且equals方法也为true的元素，如果有的话则返回，没有的话则返回null
  if ((tab = table) != null && (n = tab.length) > 0 &&
    (first = tab[(n - 1) & hash]) != null) {
    if (first.hash == hash &&
      ((k = first.key) == key || (key != null && key.equals(k))))
      return first;
    if ((e = first.next) != null) {
      if (first instanceof TreeNode)
          return ((TreeNode<K,V>)first).getTreeNode(hash, key);
      do {
        if (e.hash == hash &&
          ((k = e.key) == key || (key != null && key.equals(k))))
          return e;
      } while ((e = e.next) != null);
    }
  }
  return null;
}

移除元素

移除元素使用的是remove方法，实现如下:

public V remove(Object key) {
  Node<K,V> e;
  return (e = removeNode(hash(key), key, null, false, true)) == null ?
      null : e.value;
}

实际调用的是removeNode方法

final Node<K,V> removeNode(int hash, Object key, Object value,
                         boolean matchValue, boolean movable) {
  Node<K,V>[] tab; Node<K,V> p; int n, index;
  //先查找元素
  if ((tab = table) != null && (n = tab.length) > 0 &&
    (p = tab[index = (n - 1) & hash]) != null) {
    Node<K,V> node = null, e; K k; V v;
    if (p.hash == hash &&
      ((k = p.key) == key || (key != null && key.equals(k))))
      node = p;
    else if ((e = p.next) != null) {
      if (p instanceof TreeNode)
        node = ((TreeNode<K,V>)p).getTreeNode(hash, key);
      else {
        do {
          if (e.hash == hash &&
            ((k = e.key) == key ||
             (key != null && key.equals(k)))) {
            node = e;
            break;
          }
          p = e;
        } while ((e = e.next) != null);
      }
    }
    //如果存在的话则删除
    if (node != null && (!matchValue || (v = node.value) == value ||(value != null && value.equals(v)))) {
      if (node instanceof TreeNode)
        ((TreeNode<K,V>)node).removeTreeNode(this, tab, movable);
      else if (node == p)
        tab[index] = node.next;
      else
        p.next = node.next;
      ++modCount;
      --size;
      //这里是个空实现，主要是为了删除元素后LinkedHashMap对双向链表进行删除
      afterNodeRemoval(node);
      return node;
    }
  }
  return null;
}

序列化

序列化使用的是writeObject方法

private void writeObject(java.io.ObjectOutputStream s)
    throws IOException {
  int buckets = capacity();
  //写入threshold、loadfactor等属性
  s.defaultWriteObject();
  //写入数组大小
  s.writeInt(buckets);
  //写入元素个数
  s.writeInt(size);
  //写入哈希桶数组中每个元素
  internalWriteEntries(s);
}

反序列化

反序列化使用的是readObject方法

private void readObject(java.io.ObjectInputStream s)
    throws IOException, ClassNotFoundException {
  //获取threshold，loadfactor等属性
  s.defaultReadObject();
  //初始化必要的属性
  reinitialize();
  if (loadFactor <= 0 || Float.isNaN(loadFactor))
    throw new InvalidObjectException("Illegal load factor: " +
                                     loadFactor);
  //读取写入时的buckets
  s.readInt();      
  //读取HashMap中元素的个数          
  int mappings = s.readInt();
  if (mappings < 0)
    throw new InvalidObjectException("Illegal mappings count: " +mappings);
  else if (mappings > 0) {
    //根据元素个数确定哈希桶数组大小
    float lf = Math.min(Math.max(0.25f, loadFactor), 4.0f);
    float fc = (float)mappings / lf + 1.0f;
    int cap = ((fc < DEFAULT_INITIAL_CAPACITY) ?
               DEFAULT_INITIAL_CAPACITY :
               (fc >= MAXIMUM_CAPACITY) ?
               MAXIMUM_CAPACITY :
               tableSizeFor((int)fc));
    float ft = (float)cap * lf;
    threshold = ((cap < MAXIMUM_CAPACITY && ft < MAXIMUM_CAPACITY) ?
                 (int)ft : Integer.MAX_VALUE);
    @SuppressWarnings({"rawtypes","unchecked"})
    Node<K,V>[] tab = (Node<K,V>[])new Node[cap];
    table = tab;

    //读取每个元素，并调用putVal方法重新构建哈希桶数组
    for (int i = 0; i < mappings; i++) {
      @SuppressWarnings("unchecked")
      K key = (K) s.readObject();
      @SuppressWarnings("unchecked")
      V value = (V) s.readObject();
      putVal(hash(key), key, value, false, false);
    }
  }
}
